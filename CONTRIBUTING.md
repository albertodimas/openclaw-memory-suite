# Contributing

Gracias por tu interes en contribuir. Este repo prioriza calidad, claridad y reproducibilidad.

## Flujo de contribucion
1) Abre un issue para discutir la idea.
2) Haz un fork y crea una rama con un nombre claro.
3) Manten cambios enfocados y bien documentados.
4) Abre un PR con descripcion clara y pasos para probar.

## Estilo de codigo
- JavaScript/TypeScript limpio, sin dependencias innecesarias.
- Evita logs ruidosos en produccion.
- Prefiere configuracion explicita y defaults seguros.

## Pruebas
- Si agregas logica critica, incluye pruebas cuando sea posible.
- Documenta pasos de verificacion manual si no hay tests.

## Reportar bugs
Incluye:
- Version de OpenClaw
- OS
- Proveedor (OpenAI/Ollama/etc)
- Pasos para reproducir

## Licencia
Al contribuir, aceptas que tu codigo se publica bajo la licencia MIT del repo.
